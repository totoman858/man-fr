NOM

xawtv - un programme X11 pour regarder la télévision  

SYNOPSIS

xawtv [ options ] [ nom de la station ]  

DESCRIPTION

xawtv est une application X11 permettant de regarder la télévision avec votre machine unix. Elle utilise les contrôles Athena.  

OPTIONS

Toutes les options de la ligne de commande correspondent à une ressource, vous pouvez donc les mettre aussi dans votre fichier .Xdefaults.
-h, -help
Affiche une courte description de toutes les options de la ligne de commande.
-hwscan
Affiche une liste des périphériques disponibles.
-f, -fullscreen
Démarre en mode plein écran.
Ressource: xawtv.fullscreen (booléen).
-n, -noconf
ne lit pas le fichier de configuration ($HOME/.xawtv).
Ressource: xawtv.readconfig (booléen).
-o, -outfile base
définit le nom de base pour les fichiers de capture d'écran. Le nom du fichier sera du type "base-date-numéro.ext".
Ressource: xawtv.basename (chaîne).
-c, -périphérique périphérique
définit le périphérique video4linux (par défaut /dev/video0). Cette option désactive aussi Xvideo.
Ressource: xawtv.périphérique (chaîne).
-C, -dspdev périphérique
définit le périphérique audio (par défaut /dev/dsp).
Ressource: xawtv.dspdev (chaîne).
-vbidev périphérique
définit le périphérique vbi (par défaut /dev/vbi).
Ressource: xawtv.vbidev (chaîne).
-c, -xvport nombre
définit le port Xvideo (par défaut le premier utilisable si Xvideo est disponible). Cette option active aussi Xvideo.
Ressource: xawtv.xvport (int).
-joydev périphérique
périphérique manette de jeu à utiliser pour contrôler xawtv.
Ressource: xawtv.joydev (chaîne).
-v, -debug niveau
Etre plus explicite, le niveau peut être 0 (par défaut), 1 ou 2. Cela n'existe que pour le débogage ou pour l'analyse de problèmes.
Ressource: xawtv.debug (int).
-remote
L'affichage X11 est distant. Ceci désactive en fait le mode overlay, vous pouvez en avoir besoin si xawtv n'arrive pas à détecter automatiquement un affichage X11 distant.
Ressource: xawtv.remote (booléen).
-b, -bpp n
force xawtv à basculer dans la profondeur de couleur n. Permet de basculer entre 15/16 et 24/32 bpp. Ce paramètre est seulement renvoyé à v4l-conf(1).
Ressource: xawtv.bpp (int).
-shift n
Décale l'affichage video de n octets. Essayez ceci si l'image video n'est pas dans la fenêtre de xawtv, quelques cartes matrox anciennes sont affectées de ce problème. Ce paramètre aussi est seulement renvoyé à v4l-conf(1).
Ressource: xawtv.shift (int).
-fb
Utilise le périphérique framebuffer pour déterminer l'adresse du framebuffer (plutôt que l'extension DGA). Encore un argument renvoyé.
Ressource: xawtv.fbdev (booléen).
-xv/-noxv
Active/Désactive l'utilisation de l'extension Xvideo globalement, cad. que -noxv équivaut aux options -noxv-video et -noxv-image combinées. (voir ci-dessous).
Ressource: xawtv.xv (booléen).
-xv-video/-noxv-video
Active/Désactive l'utilisation de l'extension Xvideo pour l'overlay (ex. XvPutVideo). Le mode overlay utilise ceci. Est activé par défaut.
Ressource: xawtv.xvVideo (booléen).
-xv-image/-noxv-image
Active/Désactive l'utilisation de l'extension Xvideo pour la mise à l'échelle de l'image (ex. XvPutImage). Le mode grabdisplay utilise ceci. Est activé par défaut.
Ressource: xawtv.xvImage (booléen).
-vm/-novm
Active/Désactive l'utilisation de l'extension VidMode, est désactivé par défaut. S'il y a un mode plein écran défini dans le fichier de configuration, l'extension VidMode est activée automatiquement.
Ressource: xawtv.vidmode (booléen).
-dga/-nodga
Active/Désactive l'utilisation de l'extension DGA, activée par défaut.
Ressource: xawtv.dga (booléen).
-parallel n
utilise n instances (threads) de compression (par défaut : un).
Ressource: xawtv.parallel (int).

L'options habituelles de l'environnement comme -geometry sont aussi supportées.  

MATÉRIEL SUPPORTÉ / INTERFACES

v4l
video4linux - la API linux de capture video.
v4l2
video4linux deux - une nouvelle interface de capture video améliorée au niveau des bogues de conceptionde V4l.
bktr
Pilote pour FreeBSD / OpenBSD des cartes d'acquisition bt848/878.
Xvideo
Extention X11 pour les périphériques video. Notez que l'extension Xvideo ne supporte que le mode overlay, vous ne pouvez pas enregistrer des images ou des videos quand l'extension est utilisée. Dans un autre sens c'est le seul moyen d'obtenir une mise à l'échelle de la video (ex. plein écran sans bordures en 1024x768) si à la fois le matériel et le pilote de xfree86 le permettent. Lire README.xfree4 pour plus de détails et de conseils sur comment régler Xvideo.
 

PREMIÈRE UTILISATION

Démarrez seulement xawtv. Vérifiez alors les réglages dans le menu des options (bouton droit de la souris) et ajustez les réglages (norme TV, source video, table de fréquences). Avec les touches flèches vous pouvez contrôler le syntoniseur et rechercher des émetteurs TV. Les autres touches très utiles sont listées ci-dessous.

La touche 'E' affiche l'éditeur des chaînes où vous pouvez créer/changer vos pré-sélections pour les émetteurs TV. N'oubliez pas de cliquer sur "save" pour écrire un fichier de configuration. Vous pouvez aussi créer/changer un fichier de configuration avec n'importe quel éditeur de texte, lisez xawtvrc(5) pour une description de la syntaxe du fichier de configuration. Il y a certaines options que ne peuvent être changées qu'en éditant le fichier de configuration.

Vous pouvez aussi essayer scantv. c'est un outil en ligne de commande qui recherche les émetteurs et écrit un fichier de configuration de xawtv avec toutes les chaînes qu'il a trouvé.  

Utilisation de la souris

Le bouton gauche de la souris fait apparaître un menu avec toutes les chaînes trouvées dans le fichier de configuration. Le bouton du milieu passe à la chaîne suivante. Le bouton droit fait apparaître un menu avec beaucoup de réglages et de contrôles.

BOGUES

Les rapports de bogues avec une image attachée vont dans /dev/null sans être regardés.

xawtv a besoin d'un pilote correctement configuré. Si vous ne pouvez pas syntoniser des émetteurs alors que les réglages dans la fenêtre des options sont corrects c'est certainement un problème de pilote.

L'affichage mono/stereo n'est pas fiable de par les limitations de l'API v4l. L'API ne peut pas renvoyer le mode audio en cours, seulement une liste des modes disponibles. Xawtv essaye seulement de deviner, partant du principe que le pilote utilise le meilleur mode disponible. Suivant votre matériel ceci peut être faux. Si vous n'êtes pas sûr du fait que la stereo fonctionne réellement, veuillez syntoniser une chaîne musicale et écoutez, ne faites pas confiance à ce que xawtv dit.
