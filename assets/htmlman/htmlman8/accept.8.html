NOM

accept/reject - Accepte/refuse les travaux d'impression envoyés à une destination  

SYNOPSIS

accept destination(s)
reject [ -E ] [ -h serveur ] [ -r raison ] destination(s)  

DESCRIPTION

accept envoie l'ordre au système d'impression d'accepter les travaux d'impression vers les destinations spécifiées.

reject envoie l'ordre au système d'impression de refuser les travaux d'impression vers les destinations spécifiées. L'option -r permet de spécifier la raison du rejet des travaux d'impression. Si cette raison n'est pas fournie, la raison par défaut est « Raison inconnue ».

L'option -E force le cryptage lors de la connexion au serveur.  

COMPATIBILITÉ

Les versions CUPS d'accept et reject peuvent demander à l'utilisateur de s'identifier suivant la configuration du système d'impression, à la différence des versions System V, qui demandent le mot de passe root.
