NOM

update-modules - (re)fabrique /etc/modules.conf et /etc/chandev.conf  

SYNOPSIS

update-modules [force]  

DESCRIPTION

update-modules est un outil simple permettant de fabriquer les fichiers /etc/modules.conf et /etc/chandev.conf pour la plate-forme s390. Les outils de gestion des modules Linux utilisent un seul fichier de configuration pour configurer tous les modules. Au sein de ce fichier, un paquet peut difficilement configurer dynamiquement ses propres modules. update-modules permet de rendre cette manipulation plus aisée.

update-modules tente de résoudre le problème en fabriquant le fichier de configuration à partir de fichiers distincts, situés dans le répertoire /etc/modutils. Tous les fichiers de ce répertoire sont assemblés pour former /etc/modules.conf et /etc/chandev.conf. Il y a aussi le répertoire spécial (/etc/modutils/arch) contenant des informations dédiées aux plates-formes particulières. update-modules détermine la plate-forme et choisit le fichier à utiliser.

Après sa fabrication, l'ancien fichier de configuration est copié dans le fichier /etc/modules.conf.old ou bien dans le fichier /etc/chandev.conf.old.  

FICHIERS

Vous pouvez placer deux sortes de fichiers dans /etc/modutils, les fichiers normaux et les fichiers exécutables. Les fichiers normaux contiennent la configuration des modules standard et utilisent une syntaxe identique à celle de modules.conf(5). Les fichiers d'extension .chandev contiennent les paramètres des modules et utilisent la même syntaxe que celle de chandev(8). Le fichier spécial 0chandev.chandev contient les options des canaux de la couche périphérique. Les fichiers exécutables sont lancés et leur sortie est utilisée en tant qu'information de configuration complémentaire. Les messages d'erreurs envoyés sur la sortie d'erreur standard ne sont pas redirigés.

update-modules ignore les fichiers correspondant à l'expression rationnelle suivante :

*\(\.dpkg-[a-z]*\|~\)$
(par exemple les fichiers d'extensions .dpkg-new, .dpkg-orig, .dpkg-dist, .dpkg-old, ou ceux dont le nom se termine par ~).  

OPTIONS

force
update-modules vérifie qu'il a bien fabriqué les fichiers actuels /etc/modules.conf et /etc/chandev.conf , en s'assurant de la présence d'un marqueur particulier sur la première ligne. Si le marqueur n'est pas présent, la fabrication est abandonnée. En fournissant le paramètre force , seul un avertissement est affiché.
