NOM

pwck - vérifie l'intégrité des fichiers de mots de passe

 

SYNOPSIS

pwck [-sr] [passwd shadow]

 

DESCRIPTION

pwck vérifie l'intégrité des informations d'authentification du système. Toutes les entrées des fichiers /etc/passwd et /etc/shadow sont vérifiées pour voir si elles sont correctement formatées et si chaque champ possède une valeur valide. Il sera demandé à l'utilisateur d'effacer les entrées qui ne sont pas correctement formatées, ou s'il y a des erreurs incorrigibles.

Les vérifications sont faites pour être certain que chaque entrée possède :

- un nombre correct de champs
- un nom d'utilisateur unique
- des identifiants utilisateur et groupe valides
- un groupe primaire valide
- un répertoire personnel valide
- un interpréteur de commande valide à la connexion

Les vérifications sur le nombre correct de champs et sur l'unicité du nom d'utilisateur sont fatales. Si l'entrée n'a pas un nombre correct de champs, il sera demandé à l'utilisateur d'effacer la ligne complète. Si l'utilisateur ne répond pas par l'affirmative, toutes les autres vérifications ne seront pas effectuées. Dans le cas d'une entrée avec un nom d'utilisateur dupliqué, l'effacement sera proposé à l'utilisateur, mais les vérifications restantes seront effectuées. Toutes les autres erreurs seront indiquées sous forme d'avertissement et l'utilisateur se voit encouragé à lancer la commande usermod pour corriger les erreurs.

Les commandes qui opèrent sur le fichier /etc/passwd ne sont pas capables d'altérer les entrées corrompues ou dupliquées. Dans certaines circonstances, pwck devrait être utilisé pour effacer l'entrée fautive.

 

OPTIONS

Par défaut, pwck opère sur les fichiers /etc/passwd et /etc/shadow. L'utilisateur peut choisir des fichiers alternatifs avec les paramètres passwd et shadow. De plus, l'utilisateur peut exécuter la commande en mode lecture seule en utilisant l'option -r. Cette option fait qu'il sera répondu non à toutes les questions relatives à des modifications, sans intervention de l'utilisateur. grpck peut aussi classer les entrées dans les fichiers /etc/passwd et /etc/shadow par UID. Pour lancer la commande avec le mode de tri, passez-lui l'option -s. Aucune vérification n'est effectuée, cela effectue simplement le tri.

 

FICHIERS

/etc/passwd - informations sur les comptes utilisateurs
/etc/shadow - informations sur les mots de passe masqués
/etc/group - informations sur les groupes  

VOIR AUSSI

group(5), passwd(5), shadow(5), usermod(8)

 

DIAGNOSTICS

La commande pwck renvoie les valeurs suivantes en quittant :
0
Succès
1
Erreur de syntaxe
2
Une ou plusieurs mauvaises entrées mot de passe
3
Impossible d'ouvrir les fichiers de mots de passe
4
Impossible de verrouiller les fichiers de mots de passe
5
Impossible de mettre à jour les fichiers de mots de passe
