NOM

modinfo - Informations à propos d'un module du noyau.  

SYNOPSIS

modinfo [ options ] <module_file>  

DESCRIPTION

L'utilitaire modinfo examine le fichier objet module_file associé à un module du noyau et affiche les informations qu'il peut y recueillir.  

OPTIONS

-a, --author
Affiche l'auteur du module.
-d, --description
Affiche une description du module.
-fformat_string, --format format_string
Utilise la chaîne de format spécifiée par l'utilisateur pour extraire les valeurs de la section ELF de module_file qui contient l'information sur le module. Les remplacements consistent en un signe de pourcentage suivi du nom d'étiquette entre accolades. Un nom d'étiquette %{filename} est toujours supporté, même si le module n'a pas de section modinfo. %{kernel_version} affiche le numéro de version du noyau pour lequel le module a été compilé. %{{using_checksums} renvoie 1 si le module a des numéros de version sur les symboles et 0 ou <none> sinon.

Un nom d'étiquette %{parm} est spécial : la chaîne de format est répétée pour chaque paramètre connu du module (ce qui peut être zéro fois) et %{parm} est remplacé par la description du paramètre du modules (un paramètre sur chaque ligne).

De plus, %a, %d, %l, %n et %p peuvent servir à remplacer %{author}, %{description}, %{filename}, %{license} et %{parm}, respectivement.

-h, --help
Affiche un court message d'aide.
-l, --license
Affiche la licence du module.
-n, --filename
Affiche le nom du fichier du module.
-p, --parameters
Affiche le type de paramètres que le module accepte.
-V, --version
Affiche la version de modinfo.

Si aucune option n'est fournie, l'affichage par défaut est nom de fichier, description, auteur, licence et paramètres.
