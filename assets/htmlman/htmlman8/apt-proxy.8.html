NOM

apt-proxy - Un mandataire pour économiser de la bande passante sur les serveurs Debian.  

SYNOPSIS

apt-proxy [options] [fichier de log]

 

DESCRIPTION

apt-proxy est un script shell avancé conçu pour être lancé en dehors d'inetd, et donne un mandataire (« proxy ») intelligent, propre et cachant pour apt-get, qui parle http aux clients apt-get et http, rsync ou ftp aux serveur(s) backend(s). Il est normalement lancé sur le port 9999, principalement parce que c'est la configuration par défaut et que les gens sont fainéants.

 

OPTIONS

Les options de la ligne de commande d'apt-proxy sont :
-c <fichier de configuration>
Spécifie quel fichier est utilisé pour la sauvegarde de la configuration, par défaut /etc/apt-proxy.conf.
-l <fichier de log>
Spécifie le fichier dans lequel les journaux sont enregistrés, par défaut /var/log/apt-proxy. Le fichier de log peut aussi être spécifié en tant que premier paramètre (non optionnel) en ligne de commande.
-v, --version
Affiche la version du programme.
 

CONFIGURATION DU CLIENT

Une fois apt-proxy configuré, les utilisateurs éditent leur fichier sources.list pour le faire pointer vers le mandataire (qui utilise le protocole http pour servir les clients) comme décrit ci-dessous : 

deb http://SERVER:9999/main woody main contrib non-free
deb http://SERVER:9999/non-US woody/non-US main contrib non-free
deb-src http://SERVER:9999/main woody main contrib non-free
deb-src http://SERVER:9999/non-US woody/non-US main contrib non-free

deb http://SERVER:9999/openoffice/ woody main contrib

Le chemin devant être spécifié après le nom du serveur et le numéro de port dépend de la configuration d'apt-proxy (qui peut restreindre les chemins et envoyer des chemins différents à différents serveurs). Dans cet exemple, non-US/ et openoffice/ récupèrent finalement des fichiers de différents serveurs backends.

 

CONFIGURATION DU SERVEUR

Voir apt-proxy.conf(5) pour les détails pour configurer apt-proxy pour l'utilisation des backends proches de vous.

 

SOIN ET APPROVISIONNEMENT DE MIROIRS

apt-proxy réduit les nécessités en bande passante des miroirs Debian en synchronisant les fichiers Packages non comprimés quand c'est possible (beaucoup plus léger que de télécharger le fichier comprimé entier), en restreignant la fréquence de mise à jour des fichiers Packages, Releases et Sources à partir du backend, en gardant les transferts partiels, et en faisant seulement un unique fetch pour chaque fichier, quel que soit le nombre d'utilisateurs le demandant au mandataire.

 

FICHIERS

/etc/apt-proxy/apt-proxy.conf

 

VOIR AUSSI

apt-proxy.conf(5)

 

BOGUES

Les paquets ne sont pas comprimés en utilisant gzip --rsyncable, ce qui donne une réduction de 30 % en terme d'octets transferés pour les paquets binaires, et beaucoup plus pour les paquets sources et autres.
