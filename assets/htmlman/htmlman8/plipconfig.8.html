PLIPCONFIG

Section: Manuel de l'administrateur Linux (8)
Updated: 17 février 1995

NOM

plipconfig - Régler finement les paramètres du périphérique PLIP.  

SYNOPSIS

plipconfig interface
plipconfig interface [nibble NN] [trigger NN] [unit NN]  

DESCRIPTION

plipconfig est utilisé pour améliorer (espérons-le) les performances PLIP en modifiant les minuteries par défaut utilisées par le protocole PLIP. Les résultats dépendent du port parallèle utilisé, du câble et de la vitesse du CPU des machines à chaque extrémité de la liaison PLIP.

Si interface est le seul argument fourni, plipconfig affiche seulement l'état de cette interface. Autrement, il essaie de positionner les options données.  

OPTIONS

nibble NN
Définir le temps d'attente des quartets en microsecondes. La valeur par défaut est 3000.
trigger NN
Définir le temps d'attente de déclenchement en microsecondes. La valeur par défaut est 500.

La vitesse de PLIP peut, dans certains cas, être améliorée en diminuant les valeurs par défaut. Des valeurs trop faibles peuvent engendrer une surcharge de travail du processeur, et un mauvais temps de réponse aux interruptions, pouvant conduire à perdre des caractères sur le port ou des paquets PLIP. La modification du MTU de PLIP peut aussi affecter la vitesse de PLIP.  

NOTE

Si vous n'obtenez aucune réponse, il est fort probable que l'IRQ configurée soit erronée, et qu'elle nécessite une configuration avec ifconfig(8). Les quelques cas pour lesquels les minuteries par défaut sont trop courtes, arrivent lorsque l'on utilise de longs câbles. C'est une chose à éviter puisque le port parallèle n'est pas conçu pour supporter de longs câbles.
