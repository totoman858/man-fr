RARP

Section: Manuel de l'administrateur Linux (8)
Updated: 4 août 1997

NOM

rarp - Manipuler la table système RARP.  

SYNOPSIS

rarp [-V] [--version] [-h] [--help]
rarp -a
rarp [-v] -d nom_hôte ...
rarp [-v] [-t type] -s nom_hôte adr_matériel  

NOTE

Ce programme est obsolète. À partir de la version 2.3, le noyau Linux ne prend plus en charge RARP. Pour remplacer le daemon RARP, consultez ftp://ftp.dementia.org/pub/net-tools.  

DESCRIPTION

rarp manipule la table RARP du noyau de différentes façons. Les options de base permettent d'effacer une entrée de correspondance d'adresses et d'en définir une manuellement. Pour des besoins de débogage, le programme rarp permet aussi de vider complètement la table RARP.  

OPTIONS

-V
Afficher la version de RARP utilisée.
-v
Mode volubile.
-t type
En définissant ou lisant la table RARP, ce paramètre optionnel dit à rarp quelles classes d'entrées il doit contrôler. La valeur par défaut de ce paramètre est ether (c'est-à-dire, code matériel 0x01 pour IEEE 802.3 10Mbps Ethernet). D'autres valeurs correspondent à des technologies de réseau telles que AX.25 (ax25) et NET/ROM (netrom).
-a
--list
Afficher les entrées de la table RARP.
-d nom_hôte
--delete nom_hôte
Supprimer toutes les entrées RARP relatives à l'hôte spécifié.
-s nom_hôte adr_matériel
--set nom_hôte adr_matériel
Créer manuellement une correspondance d'adresses RARP pour l'hôte nom_hôte avec l'adresse du matériel adr_matériel. Le format de l'adresse du matériel est dépendant de la classe de matériel, mais pour la plupart, on peut considérer que la présentation classique peut être utilisée. Pour la classe Ethernet, l'adresse est donnée sur 6 octets en hexadécimal, séparés par des double-points.
 

ATTENTION

Certains systèmes (notamment des vieux Sun) supposent que l'hôte répondant à une requête RARP peut aussi fournir des services de démarrage à distance. Par conséquent, n'ajoutez jamais d'entrées RARP à la légère, à moins que vous ne souhaitiez affronter la colère de votre administrateur de réseau.
