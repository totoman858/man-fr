NOM

gzexe - Compacter sur place les fichiers exécutables.  

SYNOPSIS

gzexe [ nom ... ]  

DESCRIPTION

L'utilitaire gzexe vous permet de compacter sur place des exécutables pour qu'ils se décompactent automatiquement et s'exécutent là où vous les appelez (au prix d'une perte de performance). Par exemple, si vous exécutez « gzexe /bin/cat », il créera les deux fichiers suivants : 



    -r-xr-xr-x  1 root  bin   9644 Feb 11 11:16 /bin/cat
    -r-xr-xr-x  1 bin   bin  24576 Nov 23 13:21 /bin/cat~

/bin/cat~ est le fichier original et /bin/cat est le fichier exécutable auto-extractible. Vous pouvez supprimer /bin/cat~ une fois que vous êtes sûr(e) que /bin/cat fonctionne correctement.

Cet utilitaire est principalement utile sur les systèmes ayant de très petits disques.  

OPTIONS

-d
Décompacter les exécutables fournis au lieu de les compacter.
 

VOIR AUSSI

gzip(1), znew(1), zmore(1), zcmp(1), zforce(1)  

AVERTISSEMENTS

L'exécutable compacté est un script shell. Cela peut créer quelques trous de sécurité. En particulier, l'exécutable compacté utilise la variable d'environnement PATH pour trouver gzip et d'autres utilitaires (tail, chmod, ln, sleep).  

BOGUES

gzexe essaie de préserver les attributs du fichier original sur l'exécutable compacté, mais vous pourriez devoir les régler par vous-même dans certains cas, en utilisant chmod ou chown.
