NOM

install-catalog - Gère un catalogue centralisé SGML ou XML  

SYNOPSIS

install-catalog [ option ] action ... [ [ -d] [ --delegate] ] [ [ -a centralisé ordinaire] [ --add centralisé ordinaire]


 [ -r centralisé ordinaire]  [ --remove centralisé ordinaire]


 [ -h]  [ --help] 


 [ -v]  [ --version] ] 

 

DESCRIPTION

Ce script shell permet de traiter un catalogue centralisé référençant d'autres « Catalogues Ouverts » SGML ou XML.

Un « Catalogue Ouvert » contient un ensemble de directives définies par OASIS, principalement utilisées pour définir des équivalences entre les identifiants publics formels (FPI : Formal Public Identifiers) et les noms de fichiers réels (voir TR9401:1997 à http://www.oasis-open.org <URL:http://www.oasis-open.org>). Les « Catalogues Ouverts » se situent généralement dans /usr/share/sgml.

Un catalogue centralisé est un « Catalogue Ouvert » particulier qui inclut seulement des commentaires et des directives CATALOG pointant vers d'autres catalogues (ou des directives DELEGATE si elles sont gérées). Le catalogue centralisé peut être la clé de toutes les autres ressources SGML. Il réside généralement dans /etc/sgml.

Le catalogue centralisé peut être édité manuellement ou à travers le script install-catalog pour ajouter ou retirer des références vers d'autres catalogues ordinaires. Ce script est principalement destiné aux scripts de post-installation des paquetages logiciels.

Un exemple d'appel typique d'install-catalog est : 

      install-catalog --add \
        /usr/share/sgml/docbook/dsssl-stylesheets-1.54/catalog \
        /etc/sgml/sgml-docbook-4.0.cat

Cet appel installera dans /etc/sgml/sgml-docbook-4.0.cat une référence vers /usr/share/sgml/docbook/dsssl-stylesheets-1.54/catalog.

 

OPTIONS

Voici la liste des options qui peuvent influencer la façon de travailler d'install-catalog :


 [  [ -d]  [ --delegate] ]

Utilise les directives DELEGATE au lieu de CATALOG pour spécifier un pointeur  

ACTIONS

Voici la liste des actions qui peuvent être demandées à install-catalog :


 [  [ -a centralized ordinary]  [ --add centralized ordinary] ] 

Déclare un nouveau paquetage SGML.


 [  [ -r centralized ordinary]  [ --remove centralized ordinary] ] 

Retire un paquetage SGML précédemment installé.


 [  [ -h]  [ --help] ]

Affiche un court message d'aide et quitte.


 [  [ -v]  [ --version] ]

Affiche l'identificateur de version et quitte.
