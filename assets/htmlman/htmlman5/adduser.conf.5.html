NOM

/etc/adduser.conf - Fichier de configuration pour adduser(8) et addgroup(8).  

DESCRIPTION

Le fichier /etc/adduser.conf contient les valeurs par défaut pour les programmes adduser(8) et addgroup(8). Chaque option est de la forme option = valeur. Les simples ou doubles guillemets sont autorisés autour de la valeur. Les lignes de commentaires doivent avoir un caractère dièse (#) au début de la ligne.

Les options de configuration valides sont :

DSHELL
Le shell de login qui doit être utilisé pour tous les nouveaux utilisateurs. Par défaut c'est /bin/bash qui est employé.
DHOME
Le répertoire dans lequel doit être créé le répertoire personnel des nouveaux utilisateurs. Par défaut positionné à /home.
GROUPHOMES
Si positionné à yes, les répertoires personnels seront de la forme /home/[nomgroupe]/utilisateur. Par défaut positionné à no.
LETTERHOMES
Si positionné à yes, alors les répertoires personnels créés auront un répertoire supplémentaire qui sera l'initiale du nom d'utilisateur. Par exemple : /home/u/utilisateur. Par défaut positionné à no.
SKEL
Le répertoire contenant les fichiers de configuration des utilisateurs que l'on doit copier. Par défaut /etc/skel.
FIRST_SYSTEM_UID et LAST_SYSTEM_UID
spécifie l'intervalle fermé des UIDs utilisateurs systèmes attribués dynamiquement. Par défaut 100 - 999.
FIRST_UID et LAST_UID
spécifie l'intervalle fermé des UIDs utilisateurs réguliers attribués dynamiquement. Par défaut 1000 - 29999.
USERGROUPS
Si positionné à yes, alors un groupe sera créé pour chaque utilisateur et le répertoire personnel sera g+s. Si positionné à no, alors chaque utilisateur sera placé dans le groupe dont le GID est USERS_GID (voir en dessous). Par défaut positionné à yes.
USERS_GID
Si USERGROUPS vaut no, alors USERS_GID est le GID à tous les utilisateurs nouvellement créés . La valeur par défaut est 100.
QUOTAUSER
Si positionné à une valeur non vide, les quotas des nouveaux utilisateurs seront les mêmes que ceux de cet utilisateur. Vide par défaut.
DIR_MODE
Si positionné à une valeur valide (par exemple 0755 ou 755), les répertoires créés auront la permission spécifiée. Sinon 0755 est pris par défaut.
SETGID_HOME
Si positionné à yes, alors les répertoires personnels des utilisateurs qui possèdent leur propre groupe ( USERGROUPS=yes ) auront le bit setgid positionné. C'était le paramètre par défaut pour les versions de adduser << 3.13. Malheureusement il y a des effets de bord gênants, alors nous ne le mettrons plus par défaut. Si vous le voulez malgré tout, vous pouvez l'activer ici.
