NOM

adduser, addgroup - Ajouter un utilisateur ou un groupe au système.  

SYNOPSIS

adduser [options] [--home DIR] [--shell SHELL] [--no-create-home] [--uid ID] [--firstuid ID] [--lastuid ID] [--ingroup GROUP | --gid ID] [--disabled-password] [--disabled-login] [--gecos GECOS] utilisateur

adduser --system [options] [--home DIR] [--shell SHELL] [--no-create-home] [--uid ID] [--group | --ingroup GROUP | --gid ID] [--disabled-password] [--disabled-login] [--gecos GECOS] utilisateur

adduser --group [options] [--gid ID] groupe
addgroup [options] [--gid ID] groupe

adduser --group --system [options] [--gid ID] groupe
addgroup --system [options] [--gid ID] groupe

adduser [options] utilisateur groupe  

OPTIONS STANDARD


[--quiet] [--force-badname] [--help] [--version] [--conf FICHIER]  

DESCRIPTION

adduser et addgroup ajoutent des utilisateurs et des groupes au système en fonction des options de la ligne de commande et des informations contenues dans le fichier de configuration /etc/adduser.conf. Ce sont des interfaces plus conviviales aux programmes useradd et groupadd, se conformant à la charte Debian concernant les valeurs d'UID et de GID, créant un répertoire personnel avec une configuration suivant un modèle (squelette), lançant un script personnalisé, ainsi que d'autres fonctionnalités. adduser et addgroup peuvent être lancé de 5 manières différentes :  

Ajouter un utilisateur normal

Si cette option est appelée avec un argument sans options ou sans les options --system ou --group adduser ajoutera un utilisateur normal.

adduser va choisir le premier UID disponible depuis la gamme spécifiée pour les utilisateurs normaux dans le fichier de configuration. L'UID peut être modifié avec l'option --uid.

La gamme spécifiée dans le fichier peut être modifiée grâce aux options --firstuid et --lastuid.

Par défaut, à chaque utilisateur de Debian GNU/Linux créé est associé un groupe du même nom et id. Les groupes d'utilisateurs autorisent l'écriture sur les dossiers aux membres du groupe pour être facilement administrés en introduisant les utilisateurs appropriés dans ce nouveau groupe, en définissant qu'il appartient à tel groupe et en s'assurant que les autres utilisateurs ont un umask de 002. Si cette option est éteinte, en ayant défini le USERGROUPS avec no, Tous les GID des utilisateurs seront définis en tant que USERS_GID. Les groupes d'utilisateurs peuvent aussi être modifiés par une ligne de commande comportant l'option --gid ou --ingroup pour définir le groupe respectivement par id ou par nom.

adduser créera un répertoire personnel dépendant de DHOME, GROUPHOMES, et LETTERHOMES. Le répertoire personnel peut être modifié en ajoutant l'option --home dans la ligne de commande, ainsi que le shell en y ajoutant l'option --shell. Le "set-group-ID" bit du répertoire personnel est défini si la variable USERGROUPS vaut yes , alors tous les fichiers créés dans le répertoire personnel de l'utilisateur auront le groupe correct.

adduser va copier les fichiers depuis SKEL dans le répertoire personnel et vous demander l'information sur le finger (gecos) et le mot de passe. Le gecos peut aussi être défini avec l'option --gecos. Avec l'option --disabled-login , le compte sera créé mais sera effacé dès qu'un mot de passe sera défini. L'option --disabled-password ne définira pas de mot de passe, mais le login est toutefois possible par clefs SSH RSA par exemple.

Si le fichier /usr/local/sbin/adduser.local existe, il sera exécuté après que le compte a été défini *afin de faire une installation locale*. Les arguments passés au fichier adduser.local sont :
username uid gid home-directory.  

Ajouter un utilisateur système

S'il est appelé avec un argument sans options et l'option --system adduser ajoutera un utilisateur système. Si un utilisateur ayant un uid dans la gamme système (ou si le uid est spécifié avec ceci) existe déjà, adduser quittera silencieusement.

adduser choisira le premier UID disponible depuis la gamme spécifiée pour les utilisateurs système dans le fichier de configuration. L'UID peut être modifié avec l'option --uid.

Par défaut, les utilisateurs système sont placés dans le groupe nogroup. Pour placer le nouvel utilisateur système dans un groupe déjà existant, utilisez l'option --gid ou --ingroup. Pour placer le nouvel utilisateur dans un nouveau groupe ayant le même ID, utilisez l'option --group.

Un répertoire personnel est créé avec les même règles que les utilisateurs normaux. Le nouvel utilisateur système aura le shell /bin/false (qui peut être modifié par l'option --shell ), et a un mot de passe non défini. Les fichiers squelettes de configuration ne sont pas copiés.  

Ajouter un groupe d'utilisateur

Si adduser est appelé avec l'option --group et sans l'option --system , ou si addgroup est appelé respectivement, un groupe d'utilisateur sera ajouté.

Un GID sera choisi depuis la gamme spécifiée pour les UIDS des utilisateurs du fichier de configuration. Le GID peut être modifié avec l'option --gid Le groupe est créé sans utilisateurs.  

Ajouter un groupe système

Si adduser est appelé avec les options --group et --system, ou si addgroup est appelé de la même manière, un groupe sera créé.

Un GID sera choisi depuis la gamme spécifiée dans le fichier de configuration pour les users UIDS. Les GID peuvent être modifiés avec l'option --gid.

Ce groupe est créé, mais ne comporte aucun utilisateur.  

Ajouter un utilisateur existant dans un groupe existant

Si adduser est appelé avec deux arguments sans options, il va ajouter un utilisateur existant dans un groupe existant.

 

OPTIONS

--conf FILE
Utilise le fichier FILE à la place de /etc/adduser.conf.
--disabled-login
Ne pas lancer passwd pour définir le mot de passe. L'utilisateur ne sera pas autorisé à utiliser son compte tant que le mot de passe ne sera pas défini.
--disabled-password
Tout comme --disabled-login, mais il est possible de se loguer à travers les clefs SSH RSA, mais n'utilise pas l'authentification par mot de passe.
--force-badname
Par défaut, les noms d'utilisateur et de groupe doivent être composés d'une lettre minuscule suivie d'aucun ou de plusieurs caractères composés de lettres minuscules ou de chiffres. Cette option permet d'accepter les noms d'utilisateur enfreignant cette règle. Cette option force adduser et addgroup à être plus indulgent.
--gecos GECOS
Initialise le champ gecos pour la nouvelle entrée générée. adduser ne demandera aucune information finger si cette option est entrée.
--gid ID
À la création d'un groupe, cette options force le nouveau groupid à être le nombre donné. À la création d'un utilisateur, cette option mettra l'utilisateur dans ce groupe.
--group
Quand combiné avec --system, un groupe avec le même nom et la même ID que l'utilisateur système est créé. S'il ne sont pas combinés, --system, un groupe avec le nom donné est créé. C'est l'action par défaut si le programme est invoqué comme addgroup.
--help
Affiche de brèves instructions.
--home REPERTOIRE
Utilise REPERTOIRE comme le répertoire personnel de l'utilisateur, autre que celui spécifié dans le fichier de configuration. Si le répertoire n'existe pas, il est créé et des fichiers squelettes y sont copiés.
--shell SHELL
Utilise SHELL comme shell de démarrage de l'utilisateur, à la place de celui spécifié dans le fichier de configuration.
--ingroup GROUPE
Ajoute l'utilisateur à GROUPE à la place d'un groupe utilisateur ou du groupe par défaut défini par USERS_GID dans le fichier adduser.conf.
--no-create-home
Ne crée pas le répertoire personnel, même s'il existe.
--quiet
N'affiche pas les messages de progression.
--system
Crée un utilisateur système.
--uid ID
Force le nouveau userid à être ce nombre. adduser va s'arrêter si l'userid est déjà pris.
--firstuid ID
Modifie le premier uid dans la gamme dans laquelle l'uid est choisi.
--lastuid ID
Modifie le dernier uid dans la gamme dans laquelle l'uid est choisi.
--version
Montre la version du programme et les informations sur le copyright.
