NOM

ctluser - Contrôle des utilisateurs.  

SYNOPSIS

ctluser [ -h ] [ -m ] commande [ argument... ]  

DESCRIPTION

Ctluser envoie un message sur le canal de contrôle d'un utilisateur (luser(8)) donné. Cette commande ne peut être invoquée que par l'administrateur (sysadmin(1)).

Dans le mode de comportement normal, le message est envoyé au cerveau de l'utilisateur, qui effectue alors l'action demandée et renvoie une réponse avec un message de texte et un code de retour pour ctluser. Si le cerveau de l'utilisateur exécute correctement la commande, ctluser renverra un code de retour nul et affichera la réponse sur la sortie standard.

L'option ``-m'' sert à afficher un fichier MPEG du résultat de la commande indiquée sur la console du sysadmin(1). Remarquez que cela nécessite que l'administrateur sysadmin(1) utilise un serveur X.

Pour obtenir un résumé des commandes utilisez l'option ``-h''. Si une commande est incluse lorsque ctluser est invoqué avec l'option ``-h'', seule l'explication de cette commande sera affichée.

La liste complète des commandes est fournie ci-dessous. Notez que toutes les commandes ont un nombre fixé d'arguments. Si un paramètre est une chaîne vide, il est nécessaire de le fournir explicitement avec deux guillemets adjacents comme dans "".

addhist <entrée>
Ajouter une entrée dans le fichier des emplois de l'utilisateur.
allow raison
Ré-autoriser un utilisateur à se servir du système. La raison doit être indiquée avec le même texte que celui fourni avec la commande ``reject'' préalable. Remarquez que cette commande n'est pas encore implémentée.
begin nom
Initialiser un nouvel utilisateur avec le fichier nom indiqué. Remarquez que cette commande n'est pas encore implémentée.
cancel pid
Éliminer un processus d'un utilisateur.
changegroup groupe
Changer le groupe de l'utilisateur pour celui indiqué.
checkfile
Obliger l'utilisateur à revenir à un système de travail à base de papier et crayons.
drop
Déclencher la chute d'une lourde masse sur l'utilisateur.
flushlogs
Détruire tous les enregistrements des actions entreprises par le sysadmin(1) à l'encontre de l'utilisateur.
go raison
L'utilisateur est viré pour la raison indiquée.
hangup
Oblige l'utilisateur à se pendre.
mode
Affiche le mode de fonctionnement de l'utilisateur.
newgroup groupe
Oblige l'utilisateur à faire un groupe à lui tout seul.
pause pensée
Oblige l'utilisateur à s'interrompre pour méditer sur le sujet indiqué. Remarquez que cela peut planter certains utilisateurs.
shutdown raison
Éteindre l'utilisateur. L'effet est sensiblement le même que de le tuer.
signal sig
Envoyer le signal mentionné à l'utilisateur.
throttle raison
Oblige l'utilisateur à s'étrangler lui-même.
trace item flag
Fournit au sysadmin(1) toutes les informations sur l'activité courante de l'utilisateur.

 

BOGUES

Ctluser peut parfois pousser des utilisateurs à se plaindre à la hiérarchie. Le recours à bosskill(8) peut alors être nécessaire.
