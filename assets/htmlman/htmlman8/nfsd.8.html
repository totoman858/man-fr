NOM

rpc.nfsd - processus serveur NFS  

SYNOPSIS

/usr/sbin/rpc.nfsd [-p port] nproc  

DESCRIPTION

Le programme rpc.nfsd implémente la partie utilisateur du service NFS. Le service NFS est traité par le module du noyau nfsd.o ; le programme qui s'exécute dans l'espace utilisateur (et non dans l'espace noyau) se contente (!) de lancer un certain nombre de threads systèmes (c'est à dire que plusieurs contextes d'exécution existent dans le noyau).

Le serveur rpc.mountd fournit un ancien service nécessaire pour satisfaire les requêtes de montage par les clients NFS.

 

OPTIONS

-p port
spécifie un port diférent pour écouter les requêtes NFS. Par défaut, rpc.nfsd écoute le port 2049.
nproc
spécifie le nombre de threads du serveur NFS. Par défaut, seul un thread est lancé. Toutefois, pour des performances optimales, plusieurs threads peuvent être lancés. Le nombre optimal dépend du nombre de threads et de la charge dûe aux clients NFS, mais une bonne valeur est 8 threads. Le programme nfsstat(8) permet d'observer les effets d'une modification de cette valeur.
