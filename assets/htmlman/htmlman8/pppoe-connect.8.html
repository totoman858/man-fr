NOM

pppoe-connect - Script shell pour gérer un lien PPPoE.

 

SYNOPSIS

pppoe-connect [fichier_config]

pppoe-connect interface utilisateur [fichier_config]

 

DESCRIPTION

pppoe-connect est un script shell qui gère une connexion PPPoE en utilisant le client PPPoE espace utilisateur de Roaring Penguin. Si vous omettez fichier_config, le fichier par défaut /etc/ppp/pppoe.conf est utilisé. Si vous fournissez interface et utilisateur, alors ils auront priorité sur l'interface Ethernet et le nom d'utilisateur établis dans le fichier de configuration.

Notez que normalement, vous ne devriez pas invoquer directement pppoe-connect. À la place, utilisez pppoe-start pour établir la connexion PPPoE.

Pour commencer, pppoe-connect lit un fichier de configuration. Il établit ensuite une connexion PPPoE. Si jamais la connexion est coupée, un message est consigné dans le journal du système, et pppoe-connect rétablit la connexion. De surcroît, chaque fois que la connexion est coupée ou qu'elle ne peut être établie, pppoe-connect exécute le script /etc/ppp/pppoe-lost s'il existe et s'il est exécutable.

Le script shell pppoe-stop force pppoe-connect à sortir de sa boucle, termine la connexion et s'achève.

 

DÉTAILS TECHNIQUES

pppoe-connect utilise, à partir du fichier de configuration, les variables shell suivantes :

ETH
L'interface Ethernet connectée au modem DSL (par exemple, eth0).

USER
L'identifiant PPPoE de l'utilisateur (par exemple, b1xxnxnx@sympatico.ca).

PIDFILE
Un fichier dans lequel écrire le numéro de processus du processus pppoe-connect (par exemple, /var/run/pppoe.pid). Deux fichiers supplémentaires ($PIDFILE.pppd et $PIDFILE.pppoe) conservent respectivement les numéros de processus des processus pppd et pppoe.

En utilisant différents fichiers de configuration avec des réglages PIDFILE différents, vous pouvez gérer plusieurs connexions PPPoE. Spécifiez simplement le fichier de configuration comme argument à pppoe-start et pppoe-stop.
