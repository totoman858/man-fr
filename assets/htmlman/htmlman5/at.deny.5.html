NOM

at.allow, at.deny - déterminent qui peut soumettre des tâches via at ou batch  

DESCRIPTION

Les fichiers /etc/at.allow et /etc/at.deny déterminent quel utilisateur peut soumettre des commandes pour une exécution ultérieure via at(1) ou batch(1).

Les fichiers se présentent sous forme de listes de noms d'utilisateurs, un sur chaque ligne. Les blancs ne sont pas autorisés.

Le superutilisateur doit toujours pouvoir utiliser at.

Si le fichier /etc/at.allow existe, seuls les noms des utilisateurs qui y sont nommés, sont autorisés à utiliser at.

Si /etc/at.allow n'existe pas, /etc/at.deny est consulté.
