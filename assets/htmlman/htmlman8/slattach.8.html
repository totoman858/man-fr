NOM

slattach - Attacher une interface réseau à une ligne série.  

SYNOPSIS

slattach [-dehlLmnqv] [-c commande] [-p proto] [-s vitesse] [tty]
 

DESCRIPTION

slattach est un minuscule programme qui peut être utilisé pour attacher un terminal normal (« série ») dans un des différents modes « réseau », ceci vous permettant de l'utiliser pour des liaisons point-à-point vers d'autres systèmes.  

OPTIONS

-c commande
Exécuter « commande » lorsque la ligne est suspendue. Ceci permet de lancer des scripts ou de rétablir des connexions quand un lien est rompu.
-d
Activer l'affichage de débogage. C'est utile pour déterminer pourquoi une configuration ne fonctionne pas.
-h
Quitter lorsque la porteuse est perdue. Ceci fonctionne à la fois sur les périphériques /dev/tty et /dev/cua en contrôlant directement l'état de la porteuse toutes les 15 secondes.
-v
Activer le mode volubile. Utile pour les scripts shell.
-q
Opérer en mode silencieux - aucun message n'est généré.
-l
Créer un fichier de verrouillage pour le périphérique comme pour UUCP dans /var/lock.
-n
Équivalent à la commande « mesg n ».
-m
Ne pas initialiser la ligne en mode raw 8 bits.
-e
Quitter juste après l'initialisation du périphérique, au lieu d'attendre que la ligne soit suspendue.
-L
Activer le mode 3 fils. Le terminal est mis en mode CLOCAL, la surveillance de porteuse est désactivée.
-p proto
Définir précisément un protocole à utiliser sur la ligne. La valeur par défaut est cslip, c'est-à-dire SLIP compressé. Les autres valeurs possibles sont : slip (SLIP normal), adaptive (CSLIP/SLIP adaptatifs), ppp (Protocole Point-à-Point) et kiss (un protocole utilisé pour communiquer avec des contrôleurs de terminaux radio pour paquets AX.25). L'argument spécifique tty peut être utilisé pour refaire passer le périphérique en fonctionnement série normal. L'utilisation du mode « ppp » n'est en principe pas utile puisque ppp nécessite qu'un démon supplémentaire pppd soit actif sur la ligne. Pour les connexions « kiss », le programme axattach doit être utilisé.
-s vitesse
Définir la vitesse de la ligne, différente de la valeur par défaut.

Si aucun argument n'est donné, la ligne du terminal actuel (habituellement lié au login) est utilisée. Autrement, une tentative est effectuée pour obtenir le port du terminal indiqué, puis il est verrouillé et ouvert.  

FICHIERS

/dev/cua*
/var/lock/LCK.*
