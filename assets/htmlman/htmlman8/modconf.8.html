NOM

modconf - un utilitaire de configuration de module  

SYNOPSIS

modconf [--source [floppy|mounted]] [--target destination] [--libdir chemin] [--restrict-section section]* [--exclude-section section]* [--load-before module]* [--load-only module]* [--load-after module]*
 

DESCRIPTION

modconf est un script pour installer des modules du noyau Linux. Il peut être utilisé interactivement (mode graphique) ou à partir de la ligne de commande (mode batch).  

OPTIONS

Les options référencées avec un "*" ci-dessous sont appliquées seulement à un module. Si vous voulez les appliquer à plus d'un module, répétez alors l'option dans la ligne de commande.
--source floppy|mounted
Indique d'où les modules doivent être obtenus. floppy est utilisé lors de l'installation de GNU/Linux. mounted est utilisé sur les systèmes GNU/Linux déjà installés. C'est l'option par défaut.
--target répertoire
Spécifie le répertoire sous lequel etc/modules et etc/modules.conf
 sont localisés (par défaut : /). N'incluez pas de  « / » final après le nom du répertoire.
--libdir chemin
Indique où les fichiers eval et descr.gz sont localisés sur le système (par défaut : /usr/lib/module_help). N'incluez pas de « / » final après le nom du répertoire.
--restrict-section section*
Restreint modconf pour qu'il travaille simplement avec la section spécifiée de /lib/modules
--exclude-section section*
Force modconf à ne pas travailler avec la section spécifiée de /lib/modules

Sections disponibles à l'écriture de cette page de manuel : block cdrom fs ipv4 ipv6 misc net scsi (Il peut y en avoir d'autres si vous avez installé des paquets avec modules spéciaux, comme pcmcia-modules).

--load-before module*
indique un module qui sera inséré avant tous les autres.
--load-only module*
spécifie un module qui doit être inséré après tous ceux indiqués avec l'option --load-before . Rend le mode interactif inopérant.
--load-after module*
spécifie un module qui doit être inséré après tous ceux indiqués avec l'option --load-before .
Cette option est écrasée par la précédente.
 

EXEMPLES

Les exemples ci-dessous n'ont pas été testés.

modconf --load-only plip --load-before ip_masq_ftp --load-only appletalk
Insère les modules « ip_masq_ftp », « plip » et « appletalk », dans cet
ordre, et et ne lance pas le programme interactif.
